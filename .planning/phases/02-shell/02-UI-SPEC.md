## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Dock indicator dot gap, traffic light gap (0.6rem ≈ 9.6px — use 8px token), tooltip padding inline |
| sm | 8px | Dock item padding, Menubar item horizontal padding, context menu item vertical padding |
| md | 16px | Window titlebar horizontal padding, Dock container vertical padding, section padding |
| lg | 24px | Window body padding (content area frame), section breaks |
| xl | 32px | Large section gaps |
| 2xl | 48px | Dock icon base size |
| 3xl | 64px | Dock icon magnified approach distance boundary |

Exceptions:
- Menubar height: 24px (fixed by decision — source: CONTEXT.md)
- Dock icon base size: 48px (fixed by decision — source: CONTEXT.md)
- Dock icon max magnified size: 80px (fixed by decision — source: CONTEXT.md)
- Traffic light button size: 12px diameter (aligned with reference: 0.8rem = 12.8px, rounded to 12px)
- Traffic light gap: 8px (reference uses 0.6rem ≈ 9.6px, snapped to nearest 8px token)
- Window border-radius: 12px (aligned with reference: 0.75rem)
- Dock border-radius: 20px (aligned with reference: 1.2rem)
- Minimum touch target for Dock icons: 48px (base size already at minimum)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (window content, Dock tooltip) | 13px | 400 | 1.5 |
| Label (Menubar app name, menu items, context menu) | 13px | 500 | 1.4 |
| Clock (Menubar right — clock only, not used elsewhere) | 12px | 500 | 1.0 (single line, no wrap) |
| Caption (Dock run indicator — not text, visual only) | n/a | n/a | n/a |

Notes:
- The reference TopBar.svelte uses `font-size: 0.8rem` (12.8px) at `font-weight: 500` for the clock — normalized to 12px/500.
- 12px is used strictly for the Menubar clock. All other text elements use 13px minimum.
- The reference DockItem tooltip uses `font-size: 0.9rem` (14.4px) — normalized to 13px/400 for body label consistency.
- Heading role not needed in Phase 2 shell chrome (no headings in Menubar, Dock, or window chrome itself).
- Font family: `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` — renders SF Pro on macOS, Segoe UI on Windows, Inter-equivalent on Linux.

Source: reference TopBar.svelte (font-size, font-weight), CONTEXT.md (Menubar height 24px constrains typography), default.

### Glass Effect

All glass surfaces (Menubar, Dock, window titlebar on focused window) use:
```
background-color: hsla(<surface-hsl>, 0.4);
backdrop-filter: blur(20px) saturate(180%);
```

Source: CONTEXT.md (decision: `backdrop-filter: blur(20px) saturate(180%)`).
Reference Dock.svelte uses `blur(10px)` — override to `blur(20px)` per CONTEXT.md decision.

### Window Shadow

- Focused: `0px 8.5px 10px rgba(0,0,0,0.28), 0px 68px 80px rgba(0,0,0,0.56)`
- Unfocused: `0px 8.5px 10px rgba(0,0,0,0.115), 0px 68px 80px rgba(0,0,0,0.23)`

Source: reference Window.svelte — use verbatim.

Accent reserved for: Dock run indicator dot, focused window accent ring (dark mode only), inline link text inside app content. NOT for: buttons, hover states, generic borders, disabled states.

Selection token (`--color-selection`) reserved for: context menu item hover background only. This keeps hover states distinct from the accent color and satisfies the accent reserved-for contract.

## Component Inventory

### WindowFrame

**Layout**
- CSS Grid: `grid-template-rows: 28px 1fr` (titlebar 28px + content area)
- Titlebar: glass surface, `backdrop-filter: blur(20px) saturate(180%)`, border-bottom `1px solid var(--color-border)`
- Traffic lights: absolute positioned at `top: 8px; left: 12px`, 3 circles 12px diameter with 8px gap
- Drag handle class: `.window-drag-handle` on titlebar element (react-rnd `dragHandleClassName`)
- Content area: `<iframe>` filling 100% of remaining space, `border: none`
- Border-radius: 12px on container, overflow hidden
- Min-size: read from `manifest.minSize`, default `200×150` — source: CONTEXT.md

**States**
- Normal: shadow (unfocused variant), opacity 1
- Focused: shadow (focused variant), z-index elevated
- Maximized: `width: 100vw`, `height: calc(100vh - 24px)`, positioned at `top: 24px, left: 0`, border-radius 0
- Minimized: Framer Motion `scale(0.1)` + `translateY` to Dock icon center coordinates, `opacity: 0`, duration 300ms, ease `easeInOut`

**Animations**
- Open: Framer Motion `initial={{ scale: 0.95, opacity: 0 }}` → `animate={{ scale: 1, opacity: 1 }}`, duration 200ms, ease `easeOut`
- Close: Framer Motion `exit={{ scale: 0.95, opacity: 0 }}`, duration 150ms, ease `easeIn`
- Minimize: Framer Motion `animate={{ scale: 0.1, opacity: 0, x: dockIconX, y: dockIconY }}`, spring `{ stiffness: 300, damping: 35 }`
- Maximize/restore: CSS transition `width 0.3s ease, height 0.3s ease, transform 0.3s ease` — source: reference Window.svelte

**Staggered placement**: new window center = `(viewport.width / 2 - windowWidth / 2) + (openWindowCount * 20)`, `(viewport.height / 2 - windowHeight / 2) + (openWindowCount * 20)` — source: CONTEXT.md

**Boundary snap**: on drag end, clamp window position so titlebar (28px) is always within viewport; if out of bounds, spring back using Framer Motion `animate` to clamped position.

**Double-click titlebar**: toggles maximized state — source: CONTEXT.md

### Dock

**Layout**
- Fixed bottom center: `position: fixed; bottom: 0; left: 50%; transform: translateX(-50%)`
- Container height: 72px (48px icon + 8px padding top + 8px padding bottom + 8px for label/dot area)
- Inner rail: `display: flex; align-items: flex-end; padding: 8px; gap: 4px`
- Border-radius: 20px
- Glass: `background-color: hsla(<surface-hsl>, 0.4); backdrop-filter: blur(20px) saturate(180%)`
- Box shadow: `inset 0 0 0 0.5px rgba(255,255,255,0.2), 0 0 0 0.5px rgba(0,0,0,0.3), rgba(0,0,0,0.3) 2px 5px 19px 7px`

**Dock icon magnification**
- Base size: 48px — source: CONTEXT.md
- Max magnified size: 80px — source: CONTEXT.md
- Influence range: 3-5 adjacent icons — source: CONTEXT.md
- Spring: Framer Motion `useSpring` with `stiffness: 1300, damping: 82` (source: `.reference/macos-preact-main` DockItem.tsx — tighter spring replicates actual macOS Dock responsiveness; replaces earlier estimate stiffness:150/damping:15/mass:0.1 which was too slow)
- Distance tracking: `useRaf` loop reads `getBoundingClientRect()` per frame into a `distance` MotionValue — avoids `useTransform` closure/stale-ref pitfall
- Distance limit: `baseWidth * 6 = 57.6 * 6 = 345.6px` (baseWidth=57.6 from preact reference; icons sized via rem conversion `width / 16`)
- Width interpolation at distance 0: `baseWidth * 2` (peak); at `±345.6px`: `baseWidth`; intermediate: 7-point Framer Motion `useTransform` curve

**Run indicator dot**
- Size: 4px × 4px circle, `border-radius: 50%`
- Color: `var(--app-color-dark)` — theme-adaptive (dark on light bg, light on dark bg)
- Visibility: `opacity: var(--opacity)` driven by inline style `{ '--opacity': +isRunning }` — avoids conditional render
- Position: `margin: 0` below icon, centered horizontally

**Hover tooltip**
- Appears above icon on hover/focus-visible
- Background: `hsla(<surface-hsl>, 0.5)`, `backdrop-filter: blur(5px)`
- Padding: `8px 12px`, border-radius: 6px
- Font: 13px / 400 / `var(--color-text)`, `white-space: nowrap`
- Shadow: `rgba(0,0,0,0.3) 0px 1px 5px 2px`
- Position: `top: -35%` (relative to button, so scales with icon height)

**Right-click context menu (3 items)**
- Not running state: single item — "打开"
- Running state: "打开" / "在 Dock 中隐藏" / "关闭"
- Source: CONTEXT.md

**Launch bounce animation**
- On app launch: icon animates `translateY(0%) → translateY(-39.2%) → translateY(0%)`, spring `{ type: 'spring', duration: 0.7 }` — source: `.reference/macos-preact-main` DockItem.tsx (percentage-based, scales with icon size; replaces earlier `-10px` absolute value)
- Triggered via `useState` animateObj + Framer Motion `animate` prop (not `useAnimation`) — simpler and avoids imperative API

## Interaction Contracts

### Window Focus
- Click anywhere on a window → call `useWindowStore.focusWindow(id)` → assign highest z-index
- z-index range: 1–999 for windows; Dock at 1000; Menubar at 1001; Context menu at 1002

### Window Dragging
- Drag handle: `.window-drag-handle` (titlebar only, excluding traffic light button area)
- Implementation: react-rnd with `dragHandleClassName=".window-drag-handle"`
- Boundary: viewport edges; on drag end, clamp `rect.x` and `rect.y` so titlebar remains visible (at least 80px of titlebar must stay in viewport)

### Window Resizing
- 8-direction resize handles via react-rnd
- Min constraints from `manifest.minSize` or `{width: 200, height: 150}`
- Max constraints: viewport minus Menubar height (no upper limit on width)

### Keyboard
- No keyboard shortcuts in Phase 2 shell chrome (tinykeys used in Phase 3+)
- Tab navigation must reach traffic light buttons (focus-visible ring: `outline: 2px solid var(--color-accent); outline-offset: 2px`)

### Welcome Window (first launch)
- Check `localStorage['vidorra:welcomed']` on Shell mount
- If absent: open Welcome app window at center of viewport, set `localStorage['vidorra:welcomed'] = '1'`
- Welcome window is not in Dock — source: CONTEXT.md

## Animation Timing Reference

| Animation | Duration | Easing | Library |
|-----------|----------|--------|---------|
| Window open | 200ms | easeOut | Framer Motion |
| Window close | 150ms | easeIn | Framer Motion |
| Window minimize | 300ms | spring `{ stiffness: 300, damping: 35 }` | Framer Motion |
| Window maximize/restore | 300ms | ease (CSS transition) | CSS |
| Dock icon magnification | continuous | spring `{ stiffness: 1300, damping: 82 }` | Framer Motion useSpring + useRaf |
| Dock icon launch bounce | ~700ms | spring `{ type: 'spring', duration: 0.7 }` | Framer Motion animate prop |
| Context menu appear | 100ms | easeOut | Framer Motion |
| Run indicator dot appear | 150ms | linear | CSS transition |

Source: CONTEXT.md (minimize 300ms, spring animation); reference Window.svelte (maximize 300ms); reference DockItem.svelte (bounce 400ms sineInOut); defaults for open/close/context menu.
