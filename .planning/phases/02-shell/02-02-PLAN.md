3. Create `packages/shell/src/components/WindowFrame/WindowFrame.tsx`:
   - Import `Rnd` from `react-rnd`, `motion` from `framer-motion`, `useWindowStore`, and `TrafficLights`.
   - Props: `{ window: WindowStoreWindow }` (the extended type from the store).
   - Subscribe to store actions via selectors: `const closeWindow = useWindowStore(s => s.closeWindow)`, etc.
   - Use `Rnd` component with:
     - `position`: Always controlled from store. If maximized: `{ x: 0, y: 24 }` (below Menubar). Else: `{ x: window.rect.x, y: window.rect.y }`.
     - `size`: If maximized: `{ width: window.innerWidth, height: window.innerHeight - 24 }` (use `window.innerWidth`/`window.innerHeight` — the browser viewport). Else: `{ width: window.rect.width, height: window.rect.height }`.
     - `disableDragging={window.state === 'maximized'}`
     - `enableResizing={window.state !== 'maximized' && window.state !== 'minimized'}`
     - `dragHandleClassName="window-drag-handle"`
     - `minWidth={window.minWidth}` and `minHeight={window.minHeight}`
     - `style={{ zIndex: window.zIndex }}`
     - `onDragStop`: Clamp position so at least 80px of titlebar stays in viewport. `clampedX = Math.max(80 - rect.width, Math.min(d.x, viewportWidth - 80))`, `clampedY = Math.max(0, Math.min(d.y, viewportHeight - 28))`. Write clamped rect to store via `setWindowRect(id, { ...rect, x: clampedX, y: clampedY })`.
     - `onResizeStop`: Compute new size from delta, write to store via `setWindowRect`.
     - `onMouseDown`: Call `focusWindow(id)`.
   - Inside Rnd, wrap content in `motion.div` with:
     - `initial={{ scale: 0.95, opacity: 0 }}`
     - `animate`: When `state === 'minimized'` -> `{ scale: 0.1, opacity: 0, y: 500 }` (approximate Dock direction); else `{ scale: 1, opacity: 1, y: 0 }`.
     - `exit={{ scale: 0.95, opacity: 0 }}`
     - `transition`: For open/close `{ duration: 0.2, ease: 'easeOut' }`. For minimize `{ type: 'spring', stiffness: 300, damping: 35 }`.
     - `data-focused={String(window.focused)}`
     - CSS class: `styles.windowFrame` plus `styles.maximized` if maximized.
   - **CRITICAL (minimize without iframe reload):** Do NOT remove minimized windows from the DOM. Instead, track an internal `minimizedHidden` state. When `state === 'minimized'`, animate the minimize motion. On the `motion.div`'s `onAnimationComplete` callback, if the window is minimized, apply the CSS class `styles.minimizedHidden` (which sets `visibility: hidden; pointer-events: none`) to the outermost wrapper. When restoring from minimized (state changes back to 'normal'), remove the `minimizedHidden` class BEFORE starting the restore animation, then animate `{ scale: 1, opacity: 1, y: 0 }`. This keeps the iframe in the DOM at all times, preventing reload on restore.
   - Titlebar: `<div className={'${styles.titlebar} window-drag-handle'} onDoubleClick={() => toggleMaximize(id)}>`.
   - TrafficLights inside titlebar.
   - Content area: `<div className={styles.content}><iframe src={window.url} title={window.title} sandbox="allow-scripts allow-same-origin" /></div>`.

2. Create `packages/shell/src/components/WindowManager/WindowManager.tsx`:
   ```typescript
   import { AnimatePresence } from 'framer-motion'
   import { useWindowStore } from '../../stores/useWindowStore'
   import { WindowFrame } from '../WindowFrame/WindowFrame'
   import styles from './WindowManager.module.css'

   export function WindowManager() {
     const windows = useWindowStore((s) => s.windows)

     return (
       <div className={styles.windowArea}>
         <AnimatePresence>
           {windows.map((win) => (
             <WindowFrame key={win.id} window={win} />
           ))}
         </AnimatePresence>
       </div>
     )
   }
   ```
   - **CRITICAL:** Do NOT filter out minimized windows. Render ALL windows from the store. Minimized windows are hidden via `visibility: hidden` in WindowFrame (handled by WindowFrame's `onAnimationComplete` callback), NOT by removing them from the render list. This prevents iframe reload on restore (RESEARCH.md anti-pattern #3).
   - `AnimatePresence` enables exit animations on `WindowFrame` when window is closed (removed from store).
   - Each `WindowFrame` must have `key={win.id}` for AnimatePresence to track.
   - The container has `pointer-events: none` — the Rnd wrapper inside WindowFrame has `pointer-events: auto` by default (react-rnd sets this).
