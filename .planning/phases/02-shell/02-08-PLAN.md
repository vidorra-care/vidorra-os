## Goal

When a window is minimized, the shrink/fade animation should fly toward the corresponding Dock icon's position rather than straight down.

---

## Approach

1. **DockItem**: expose a ref on the icon image, register its `getBoundingClientRect` in a shared store or via a callback
2. **WindowFrame**: read the target coordinates and use them in `animateTarget`

Since this requires cross-component coordinate sharing, the cleanest approach without adding a new store:
- Add a Zustand slice `dockIconPositions: Record<appId, {x, y}>` to `useWindowStore` (or a new `useDockStore`)
- `DockItem` registers its icon center on mount + resize
- `WindowFrame` reads `dockIconPositions[win.appId]` for minimize target

### Changes

**New: `packages/shell/src/stores/useDockStore.ts`**

```ts
import { create } from 'zustand'

interface DockStore {
  iconPositions: Record<string, { x: number; y: number }>
  setIconPosition: (appId: string, pos: { x: number; y: number }) => void
}

export const useDockStore = create<DockStore>((set) => ({
  iconPositions: {},
  setIconPosition: (appId, pos) =>
    set((s) => ({ iconPositions: { ...s.iconPositions, [appId]: pos } })),
}))
```

**DockItem.tsx** — register icon position on mount + after magnification:

```tsx
import { useDockStore } from '../../stores/useDockStore'

// In DockItem component body:
const setIconPosition = useDockStore((s) => s.setIconPosition)

// Update position on each RAF tick (icon moves during magnification):
useRaf(() => {
  const el = imgRef.current
  const mouseXVal = mouseX.get()
  if (el && mouseXVal !== null) {
    const rect = el.getBoundingClientRect()
    distance.set(mouseXVal - (rect.left + rect.width / 2))
    // Update dock store with current icon center
    setIconPosition(app.id, {
      x: rect.left + rect.width / 2,
      y: rect.top + rect.height / 2,
    })
    return
  }
  distance.set(beyondLimit)
  // Update with static position when mouse not hovering
  const el2 = imgRef.current
  if (el2) {
    const r = el2.getBoundingClientRect()
    setIconPosition(app.id, { x: r.left + r.width / 2, y: r.top + r.height / 2 })
  }
}, true)
```

**WindowFrame.tsx** — use dock icon position as minimize target:

```tsx
import { useDockStore } from '../../stores/useDockStore'

// In WindowFrame component body:
const iconPositions = useDockStore((s) => s.iconPositions)
const dockTarget = iconPositions[win.appId]

const animateTarget = isMinimized
  ? {
      scale: 0.1,
      opacity: 0,
      x: dockTarget
        ? dockTarget.x - (win.rect.x + win.rect.width / 2)
        : 0,
      y: dockTarget
        ? dockTarget.y - (win.rect.y + win.rect.height / 2)
        : 500,
    }
  : { scale: 1, opacity: 1, x: 0, y: 0 }
```

Note: The `x`/`y` in Framer Motion `animate` on a `motion.div` are relative offsets from current position, not absolute coordinates.

Also update `transition` for minimize to include `x` and `y` spring:
```tsx
const transition = isMinimized
  ? { type: 'spring' as const, stiffness: 300, damping: 35 }
  : { duration: 0.18, ease: 'easeOut' as const }
```
