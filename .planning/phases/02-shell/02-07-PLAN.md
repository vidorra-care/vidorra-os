## Task 2: Fix Dock bounce animation (reference keyframe approach)

### File
- `packages/shell/src/components/Dock/DockItem.tsx`

### Root cause

The current two-state bounce (`bounceUp ? '-39.2%' : '0%'`) plays spring for the up leg then instantly resets. The reference uses a keyframe array `['0%', '-39.2%', '0%']` on `motion.span` with `{ type: 'spring', duration: 0.7 }`.

In Framer Motion v12, keyframe arrays work correctly when the `animate` value is an **object** (not inline). The reference stores `{ translateY: ['0%', '-39.2%', '0%'] }` in state.

### Fix

Match reference exactly — use `animateObj` state pattern:

```tsx
const [animateObj, setAnimateObj] = useState({ y: ['0%', '0%', '0%'] })

// In motion.span:
<motion.span
  onTap={() => setAnimateObj({ y: ['0%', '-39.2%', '0%'] })}
  initial={false}
  animate={animateObj}
  transition={{ type: 'spring', duration: 0.7 }}
>
```

Remove: `bounceUp` state, `onAnimationComplete` handler.
