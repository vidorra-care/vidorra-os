# Phase 5 — UI Design Contract: Built-in Apps

> Visual and interaction contract for App Store, Settings, Calculator, and Welcome.
> Generated by gsd-ui-researcher. Consumed by gsd-planner, gsd-executor, gsd-ui-auditor.

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none — hand-crafted CSS Modules | Codebase scan: no components.json, no tailwind.config |
| Preset | not applicable | — |
| Component library | none — reuse Shell components where possible (ContextMenu) | RESEARCH.md "Don't Hand-Roll" |

## Token System (Existing — Must Mirror in All Apps)

All four built-in apps run inside iframes with isolated CSS scope. Each app must redeclare the following tokens to match the Shell. Do NOT import or assume shell CSS is inherited.

### Light (default)

```css
body,
body[data-theme='light'] {
  --app-color-primary:           hsl(211, 100%, 50%);   /* macOS blue */
  --app-color-primary-hsl:       211, 100%, 50%;
  --app-color-primary-contrast:  hsl(240, 24%, 100%);   /* white text on blue */

  --app-color-dark:              hsl(240, 3%, 11%);     /* near-black */
  --app-color-dark-hsl:          240, 3%, 11%;
  --app-color-dark-contrast:     hsl(240, 24%, 100%);

  --app-color-light:             hsl(240, 24%, 100%);   /* white */

  --app-color-light-contrast:    hsl(0, 0%, 11%);

  --app-font-family: -apple-system, BlinkMacSystemFont, 'Inter',
    'Helvetica Neue', 'Helvetica', 'Arial', sans-serif;

  /* Grey scale (HSL triplets for use with hsla()) */
  --app-color-grey-50-hsl:  0, 0%, 98%;

  --app-color-grey-200-hsl: 0, 0%, 93%;
```

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicators, inline padding |
| sm | 8px | Compact element padding (context menu items, card internal) |
| md | 16px | Default content padding, between-card gaps |
| lg | 24px | Section padding within panels |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (Welcome hero area) |
| 3xl | 64px | Page-level vertical padding (Welcome centered layout) |

Exceptions:
- Window titlebar height: 2.5rem (40px) — inherited from WindowFrame, not overridden by apps
- Dock icon base size: 48px (SHELL-04 spec) — no app changes this
- Drag trash zone minimum touch target: 44px tall — accessibility minimum for a droppable region
- App Store card grid gap: 16px (md) — uniform spacing for visual consistency

## Typography

All sizes in px; apps use `font-family: var(--app-font-family)` declared in their own CSS.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 | App Store card descriptions, Settings panel body text, error messages |
| Label | 13px (0.8125rem) | 500 (medium) | 1.4 | Button labels, nav sidebar items, modal field labels, window title (Shell sets 0.8rem/500) |
| Heading | 20px (1.25rem) | 600 (semibold) | 1.2 | App Store toolbar title, Settings section headings, Welcome "Vidorra OS" brand |
| Display | 28px (1.75rem) | 600 (semibold) | 1.1 | Welcome hero headline only |

Weights used: 400 (regular) and 600 (semibold). Weight 500 (medium) is permitted only for window-chrome labels to match existing Shell patterns; app content uses 400/600 only.

Letter spacing: 0.3–0.4px on small labels (matching existing Shell `.menuButton` and `.item` patterns).

## Color

60/30/10 split applied per app, always adaptive to light/dark via `body.dark` class:

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `rgba(255,255,255,0.8)` — `var(--color-bg)` | `rgba(30,30,30,0.85)` | App background, Calculator button grid base |
| Secondary (30%) | `rgba(245,245,245,0.92)` — `var(--color-surface)` | `rgba(40,40,40,0.92)` | App Store cards, Settings sidebar, Settings right panel, modal background |
| Accent (10%) | `hsl(211,100%,50%)` — `var(--app-color-primary)` | same (blue is constant) | Specific elements only — see below |
| Destructive | `hsl(0,74%,50%)` = `#d93025` (standard red) | same | Uninstall button, trash zone active highlight |

**Accent reserved for:**
- "Install from URL" button label text and border (App Store toolbar)
- Active/selected state on Settings sidebar nav item
- Calculator `=` (equals) button background
- Focus ring on interactive controls (0.25rem box-shadow at 40% opacity)
- Wallpaper thumbnail selected indicator (border 2px solid)

Accent is NOT used for: card hover states, general text links, secondary buttons, informational labels.

**Glass surface pattern** (inherited from Shell — use consistently):
- Surface background: `hsla(var(--app-color-light-hsl), 0.3)` with `backdrop-filter: blur(12px)`
- Card/panel elevated background: `hsla(var(--app-color-light-hsl), 0.5)`
- Dark mode double-border: `inset 0 0 0 0.9px hsla(var(--app-color-dark-hsl), 0.3), 0 0 0 1px hsla(var(--app-color-light-hsl), 0.3)`

**Destructive state — Trash Zone:**
- Idle: low-opacity grey icon, `rgba(0,0,0,0.15)` background
- Hover/drag-over: `rgba(217,48,37,0.15)` background tint + destructive red icon

## App-by-App Layout Specs

### App Store

Default window size: 800 × 560px (from built-in-apps.json defaultSize — adjust if not set yet).
Min window size: 480 × 360px.

Layout structure:
```
[ Toolbar: 48px height ]
  - Left: "App Store" heading (20px/600)
  - Right: "Install from URL" button (accent text + border, 8px vertical padding, 16px horizontal)
[ Content area: fill remaining height ]
  Grid view:  AppGrid — CSS auto-fill grid, 160px min card width, 16px gap, 16px padding
  Detail view: Full-width card — 72px icon left + content right, Uninstall button bottom-right
[ TrashZone: 48px height, full width, bottom of content area ]
  - Shows only when a card is being dragged
  - Idle: hidden (opacity 0, pointer-events none)
  - Drag active: fades in (opacity 1, transition 150ms)
```

App Card anatomy:
- Container: 160px min-width, 100% in grid cell; 12px padding; 8px border-radius; `var(--color-surface)` background
- Icon: 56px × 56px `<img>`, centered
- App name: 14px/600, centered, max 2 lines, ellipsis overflow
- Version: 12px/400, `var(--color-text-secondary)`, centered

### Settings

Default window size: 700 × 500px.
Min window size: 400 × 300px.

Layout structure:
```
[ Two-column layout — full window height ]
  Left sidebar: 200px fixed width, var(--color-surface) background, 8px padding
    Nav items: 36px height, 8px horizontal padding, 8px border-radius on hover/active
    Active item: accent background `hsla(var(--app-color-primary-hsl), 0.15)`, accent text color
  Right panel: fill remaining, 24px padding, var(--color-bg) background
    Section heading: 20px/600, margin-bottom 16px
    Controls: stacked vertically, 16px gap
```

General panel (theme toggle):
- Segmented control: 3 buttons side-by-side (Light | Dark | Auto)
- Each button: 80px width, 32px height, 13px/500 label
- Selected: `var(--app-color-primary)` background, white text
- Unselected: `var(--color-surface)` background, `var(--color-text)` text
- Container border-radius: 8px (pill container)

Wallpaper panel:
- 3 thumbnails in a row, each 120px × 80px, 8px border-radius
- Selected: 2px solid `var(--app-color-primary)` border
- Unselected: 1px solid `var(--color-border)` border

### Calculator

Default window size: 320 × 480px.
Min window size: 280 × 400px. No resize beyond max 400 × 600px recommended (fixed-ratio feel).

Layout structure:
```
[ Display area: top ~35% of window height ]
  - Right-aligned expression string: 14px/400, var(--color-text-secondary)
  - Right-aligned result: 48px/300 (thin weight for numeric display), var(--color-text)
[ Button grid: bottom ~65% of window height ]
  4 columns × 5 rows
  Each button: equal flex, 1px gap between buttons
```

Button color mapping (macOS Calculator style, light mode):
| Button Group | Background | Text |
|--------------|------------|------|
| Digit (0–9, .) | `rgba(255,255,255,0.8)` | `rgba(0,0,0,0.85)` |
| Function (AC, +/-, %) | `hsla(var(--app-color-grey-300-hsl), 1)` | `rgba(0,0,0,0.85)` |
| Operator (+, -, *, /) | `var(--app-color-primary)` | white |
| Equals (=) | `var(--app-color-primary)` | white |

Dark mode button colors (`body.dark` or `@media (prefers-color-scheme: dark)`):
| Button Group | Background |
|--------------|------------|
| Digit | `rgba(80,80,80,0.9)` |
| Function | `rgba(55,55,55,0.9)` |
| Operator / Equals | `var(--app-color-primary)` (unchanged) |

Theme detection: `@media (prefers-color-scheme: dark)` — no SDK, no KernelBus. CSS-only.

### Welcome

Default window size: 480 × 360px.
Min window size: 360 × 280px. Non-resizable feel (fixed single-purpose page).

Layout structure:
```
[ Full-screen centered flex column, gap 24px, padding 48px ]
  App icon: 80px × 80px (Vidorra OS icon)
  Display headline: 28px/600, "Welcome to Vidorra OS"
  Body: 16px/400, centered, max-width 320px, var(--color-text-secondary), line-height 1.5
  "Get Started" button: accent background, white text, 16px horizontal padding, 44px height, 8px border-radius
```

## Interaction Contracts

### App Store

| Interaction | Behavior |
|-------------|----------|
| Click "Install from URL" | Opens InstallModal as fixed overlay (not native dialog) |
| Install modal: type URL → click Install | Calls `appRegistry.install(url)`; shows inline loading state on button ("Installing..."); success clears modal; error shows inline below input |
| Click app card | Navigates to detail view via React state (`setView('detail')`) — no URL change |
| Detail view: click "All Apps" | Returns to grid view (`setView('grid')`) |
| Right-click app card | Shows ContextMenu with items: "Open", "Uninstall App" |
| Drag app card to TrashZone | TrashZone fades in on drag start; red highlight on drag-over; `appRegistry.uninstall(id)` on drop; list refreshes |
| Uninstall button (detail page) | Calls `appRegistry.uninstall(id)` immediately; navigates back to grid |

### Settings

| Interaction | Behavior |
|-------------|----------|
| Click sidebar nav item | Switches right panel content via React state |
| Click theme segment (Light/Dark/Auto) | Calls `themeEngine.setMode(value)` immediately; no confirm needed |
| Click wallpaper thumbnail | Writes `localStorage['vidorra:wallpaper']` = image path; adds selected border; Desktop updates via storage event |

### Calculator

| Interaction | Behavior |
|-------------|----------|
| Digit press | Appends to current expression string |
| Operator press | Appends operator to expression |
| "=" press | Evaluates expression with correct operator precedence; displays result |
| "AC" press | Clears expression and result to "0" |
| "+/-" press | Negates current operand |
| "%" press | Divides current operand by 100 |
| Invalid expression | Displays "Error" in result area; next digit press clears |

### Welcome

| Interaction | Behavior |
|-------------|----------|
| Page load | `app.ready()` called → iframe becomes trusted |
| Click "Get Started" | Sets `localStorage['vidorra:welcomed'] = '1'`; calls `app.window.close()` |

## Accessibility Contracts

| Requirement | Target | Rationale |
|-------------|--------|-----------|
| Interactive controls minimum size | 44 × 44px | WCAG 2.5.5 touch target |
| Focus ring | 0.25rem box-shadow `hsla(var(--app-color-primary-hsl), 0.4)` | Matches Shell ActionCenter pattern |
| Keyboard navigation | Tab through all interactive elements | Modal must trap focus while open |
| Drag fallback | Right-click "Uninstall App" always available | Drag is not keyboard-accessible |
| Color contrast | Body text on surface backgrounds: minimum 4.5:1 | `rgba(0,0,0,0.85)` on `rgba(255,255,255,0.8)` = ~9:1 ✓ |
| ARIA labels | TrashZone: `aria-label="Uninstall drop zone"`, InstallModal: `role="dialog" aria-modal="true"` | Screen reader announcement |

## Animation Contracts

Use Framer Motion where animations are needed (available as project dependency). CSS transitions for micro-interactions.

| Element | Animation | Spec |
|---------|-----------|------|
| InstallModal open | Fade + scale in | `opacity: 0→1, scale: 0.95→1`, duration 150ms ease-out |
| InstallModal close | Fade out | `opacity: 1→0`, duration 100ms ease-in |
| TrashZone reveal | Fade in on drag start | `opacity: 0→1`, duration 150ms |
| TrashZone active | Background tint transition | CSS `transition: background-color 100ms` |
| App Store card hover | Lift shadow | CSS `transition: box-shadow 100ms`, add `0 4px 12px rgba(0,0,0,0.15)` |
| Settings panel switch | Crossfade | Framer Motion `AnimatePresence`, `opacity: 0→1`, duration 120ms |
| Calculator button press | Scale down | CSS `transform: scale(0.95)`, duration 80ms, `active` pseudo-class |
| Welcome "Get Started" hover | Accent darken | CSS `filter: brightness(0.9)`, duration 100ms |

All animations must complete in under 300ms. No animation may block user interaction.
